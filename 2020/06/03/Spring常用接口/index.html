Spring常用接口

发表于 2020-06-03 分类于 Spring

Spring的核心就是Bean管理, 关于Bean定义了很多可扩展性的接口, 通过这些扩展接口可以实现很多功能.

Spring Bean生命周期

1. BeanFactory

这是Spring最核心的接口, 定义了获取bean以及bean基础属性的相关方法, 此接口需要重写的场景非常少, 一般在程序中注入/获取使用即可. 阅读BeanFactory相关源码可以从DefaultListableBeanFactory开始.

2. ApplicationContext

应用上下文(ApplicationContext)建立在BeanFactory基础之上提供更多面向应用的功能. 可以简单理解为BeanFactory是Spring框架的基础设施, 面向Spring本身. 而ApplicationContext面向使用Spring框架的开发者, 几乎所有的应用场合都可以直接使用ApplicationContext而非底层的BeanFactory. 常用的实现类为AnnotationConfigApplicationContext, FileSystemXmlApplicationContext.

3. Aware接口

在Spring中定义了一系列的Aware接口, 在Spring容器启动之后会回调这部分接口.

3.1 ApplicationContextAware

由类名可知, 继承该类后可以获取ApplicationContext对象.

3.2 BeanFactoryAware

由类名可知, 继承该类后可以获取BeanFactory对象.

3.3 BeanNameAware

由类名可知, 继承该类后可以获取容器中配置的Id属性.

3.4 MessageSourceAware

由类名可知, 继承该类后可以获取MessageSource对象.

3.5 ApplicationEventPublisherAware

由类名可知, 继承该类后可以获取ApplicationEventPublisher对象.

3.6 ResourceLoaderAware

由类名可知, 继承该类后可以获取ResourceLoader对象.

4. InitializingBean

当需要在bean属性设置完成后做些特殊的处理, 可以继承该类实现. 效果等同于init-method配置与@PostConstruct注解. 三种方式的执行顺序为: 注解 -> InitializingBean接口 -> init-method配置.

5. DisposableBean

与InitializingBean相反, 当需要在bean销毁之前做些特殊的处理, 可以继承该类实现. 效果等同于destroy-method配置与@PreDestory注解. 三种方式的执行顺序为: 注解 -> DisposableBean接口 -> destroy-method.

6. BeanFactoryPostProcessor

该接口是实现Spring容器功能扩展的重要接口. 该接口下只有一个方法, 在容器中bean实例化之前回调该方法, 在postProcessBeanFactory方法中, 我们可以获取Bean的所有配置信息并且可以修改任意属性.

7. BeanPostProcessor

该接口主要在容器进行bean实例化过程中进行回调. postProcessBeforeInitialization方法在bean初始化方法(PostConstruct注解, InitializingBean与init-method定义的方法)调用前被调用. postProcessAfterInitialization方法在bean初始化完成后进行调用.

8. InstantiationAwareBeanPostProcessor

该接口主要在容器进行bean实例化过程中进行回调. 与BeanPostProcessor不同的是, 该类的三个方法会在BeanPostProcessor调用前被调用.

  1. postProcessBeforeInstantiation在bean实例化之前调用.
  2. postProcessAfterInstantiation在bean实例化之后与注入属性之前调用
  3. postProcessProperties在postProcessAfterInstantiation执行并返回true之后调用, 该方法返回的PropertyValues为该bean进行属性赋值.
    后续的两个方法就是BeanPostProcessor中的方法, 调用时机与上面的BeanPostProcessor一致. 两者的主要区别是:
  4. BeanPostProcessor在bean实例化之后进行调用, 继承改接口可以对bean初始化进行定制化处理.
  5. InstantiationAwareBeanPostProcessor在整个bean实例化过程中都会被调用, 主要用于替换bean的默认创建方式, 比如aop通过扩展接口生成代理对象. 在日常使用中, 更推荐使用InstantiationAwareBeanPostProcessorAdapter.
    这部分更多的代码请参考Spring源码(AbstractAutowireCapableBeanFactory#createBean(String, RootBeanDefinition, Object[]))
